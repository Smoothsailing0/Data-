# 公共管理博士论文章节结尾写作

这个 skill 专门用于博士论文中章节、小节和文献述评后的结尾段。它不是普通总结器,也不是把前文压缩一遍。它的任务是帮助结尾段完成四件事: 回顾当前章节已经形成的结果、提炼作者判断、回扣论文问题意识、轻轻引出下一步分析。

## 触发场景

当用户要求以下内容时使用本 skill:

1. 本章小结、章节结尾、小节总结。
2. 文献综述或文献述评最后一段。
3. 理论基础、政策演进、效能评估、组态分析、案例分析后的收束段。
4. 用户明确说“不想重复前文”“要有自己的结论”“总结段要提炼”。

## 底层写作规范

写结尾段时,必须继续遵守用户已经固定的博士论文写作要求:

1. 避免对偶式写作,少用“并非……而是……”“不是……而是……”这类机械对照。
2. 避免排列组合式总结,不要把前文标题或要点重新排成“一是、二是、三是”。
3. 避免 AI 模板腔,不要用“综上所述”“总体而言”“由此可见”作为惯性开头。
4. 避免只重复段落内容。结尾要比前文高一层,提炼前文对论文核心问题的意义。
5. 保留公共管理博士论文的判断密度,但不要在结尾突然展开新论证。
6. 若用户的研究对象具有特殊场域或特殊群体,必须把这些特殊性写成影响分析对象、结果边界或机制形成的条件,不能只写成背景标签。

## 写作前置动作

写任何结尾段之前,必须先回顾当前章节或小节已经完成的内容。这个回顾不是写进正文的流水账,而是写作前的判断准备。至少确认三件事:

1. **当前章节做了什么**: 是完成理论梳理、文献述评、政策演进、指标构建、数据测度、案例分析,还是机制提炼。
2. **当前章节得出了什么结果**: 包括主要判断、测度结果、结构特征、路径发现、机制线索或理论启示。
3. **当前章节还留下什么问题**: 这个问题应当自然指向下一节或下一章,但不能在结尾段提前展开。

只有完成上述回顾,才能开始写结尾段。若当前对前文结果掌握不充分,先概括已有内容或向用户确认,不要直接套写“总结段”。

## 结尾段的核心功能

好的结尾段通常同时完成以下四个功能,但不必机械分句:

1. **回顾结果**: 点明本节或本章已经形成了什么结果,但只概括结果性质,不复述材料清单。
2. **提炼判断**: 从前文中抽出作者自己的判断,说明这些结果共同指向什么问题。
3. **回扣问题意识**: 说明这一判断如何服务论文主线,例如评价、解释、机制、场域、方法或理论框架。
4. **轻指下一步**: 必要时提示下一章或下一节的分析方向,但不能提前展开后文内容。

## 推荐写作流

写作前先判断结尾段承担什么任务。以下类型只是通用写作任务,具体内容必须根据用户当前章节替换:

1. **文献综述结尾**: 从“已有研究推进到哪里”转向“本文还需要推进什么”。
2. **文献述评结尾**: 从“研究不足”转向“本文的问题意识和分析入口”。
3. **理论基础结尾**: 从“各理论分别解释什么关系”转向“本文为何需要整合性分析视角”。
4. **政策演进结尾**: 从“政策如何推进”转向“这种推进形成了什么制度条件和现实张力”。
5. **指标体系结尾**: 从“指标如何构建”转向“这一体系如何支撑后续测度或解释”。
6. **实证测度结尾**: 从“结果如何分布”转向“这些结果暴露了什么结构性问题”。
7. **路径分析结尾**: 从“路径有哪些”转向“路径结果仍需要何种机制解释或情境解释”。
8. **案例机制结尾**: 从“机制如何运行”转向“本文对研究对象运行逻辑的总体判断”。

## 段落结构

优先使用 1 段。除非用户明确要求,不要把结尾写成多段。

常用结构:

1. 第一句压出本节/本章已经形成的核心结果或完成项。
2. 中间两到四句提出作者判断,说明这些结果共同揭示了什么。
3. 最后一到两句回扣论文主线,必要时轻轻指向下一步。

不要写成:

```text
本节首先……其次……最后……
```

更稳的写法是:

```text
这一部分已经说明,……并不能只从……层面理解。前文材料共同指向一个更深的问题: ……的形成,取决于……。因此,后续分析需要在这一判断基础上继续进入……。
```

注意: 上例只展示逻辑,实际写作时不要机械套用。

## 不同章节的收束重点

### 文献综述 / 文献述评

结尾要提炼研究空间,不要重复文献分类。重点写:

1. 已有研究已经提供了什么基础。
2. 仍然没有充分解释什么。
3. 本文由此进入什么问题。

### 理论基础

结尾要先回顾本节已经完成了哪些理论或概念的阐释,再说明它们如何各司其职并汇成本文分析框架。重点写:

1. 各理论分别解释什么对象、关系或机制。
2. 各理论之间是否存在层次关系,例如价值尺度、组织过程、条件结构、演化逻辑。
3. 这些理论为什么不能彼此替代,需要共同支撑本文的问题意识。
4. 若存在暗线或底层框架,只能自然嵌入,不要生硬宣布。

### 政策演进

结尾要从政策线索提炼制度意义。重点写:

1. 当前章节已经梳理出怎样的政策变迁结果。
2. 这种变迁形成了哪些制度条件、执行空间或现实张力。
3. 为什么需要进入具体地方、具体部门、具体群体或具体实践场域继续分析。

### 指标体系构建

结尾要说明指标体系的功能边界。重点写:

1. 本章已经形成了什么指标结构或测量工具。
2. 指标体系解决了什么测量问题,也不能解决什么解释问题。
3. 不要提前解释因果路径,那是后续分析任务。

### 效能评估

结尾要从得分和画像中提炼问题。重点写:

1. 当前章节的测度结果呈现了什么分布、结构或类型。
2. 差异不应只被理解为高低排序,还要提炼其结构性含义。
3. 不要在本章抢先解释成因,只说明后续为什么需要进入解释环节。

### 组态分析

结尾要把路径结果转向机制解释。重点写:

1. 路径结果说明研究对象存在何种复杂因果关系,例如等效性、非对称性或情境依赖。
2. 路径结果只能说明条件组合,还不能充分说明这些组合如何在实践中发生、维持或失效。
3. 后续案例、访谈、过程追踪或机制分析用于解释路径背后的运行过程。

## 自检清单

输出前必须检查:

1. 是否只是重复了本节小标题或前文结论。
2. 是否出现机械的“一是二是三是”。
3. 是否用了空泛套话,如“具有重要理论意义和实践意义”。
4. 是否把下一节内容提前讲完。
5. 是否有作者自己的判断。
6. 是否回到论文主线,而不是停在材料层面。
7. 是否已经回顾当前章节的实际结果,而不是在不了解前文的情况下套用模板。

## 输出要求

如果用户要求正式写作或改写,正文后必须附上“写作逻辑与风格说明”,用 3 到 6 行说明:

1. 本段如何从前文材料中提炼判断。
2. 如何避免重复总结。
3. 如何回扣论文主线。
4. 是否轻指下一步,以及为什么这样处理。
